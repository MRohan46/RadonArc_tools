Decimal to Octal Conversion

Division Method

Repeatedly divide by 8 and read remainders from bottom to top. This is the standard method for converting from decimal to any base.

Unix Permissions

Octal is used in chmod because each digit (0-7) represents permissions for owner, group, and others. Each permission set is 3 bits: read(4), write(2), execute(1).

Common Questions

How do I convert decimal to octal manually?

Divide the decimal number by 8 repeatedly, writing down the remainders (0-7). Read the remainders from last to first. For example, 493 ÷ 8 = 61r5, 61÷8=7r5, 7÷8=0r7 → 755.

What is 255 in decimal in octal?

255 in decimal = 377 in octal. This is because 255 ÷ 8 = 31r7, 31÷8=3r7, 3÷8=0r3 → 377.

Why is octal used for file permissions?

Because each octal digit perfectly represents 3 permission bits: read(4), write(2), execute(1). Sum them: 7=rwx, 6=rw-, 5=r-x, 4=r--, 3=wx, 2=w-, 1=x, 0=---.

What's the largest decimal number I can convert?

This converter handles numbers up to 2⁵³-1 (about 9 quadrillion), which covers all practical use cases including Unix permissions and programming.

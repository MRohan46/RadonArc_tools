The 4-Bit Rule

Each hexadecimal digit represents exactly 4 binary digits (bits). This 1:4 relationship makes conversion straightforward: replace each hex digit with its 4-bit binary equivalent.

Common Questions

How do I convert hex to binary manually?

Take each hexadecimal digit and replace it with its 4-bit binary equivalent using the table. For example, hex "A3" becomes "1010 0011" because A=1010 and 3=0011.

What is FF in hex in binary?

FF in hex = 1111 1111 in binary (eight 1s). This represents 255 in decimal and is the maximum value for a single byte.

Why does each hex digit become 4 binary digits?

Because 2⁴ = 16, exactly matching the 16 possible values of a hex digit (0-15). This makes hex the perfect compact representation of binary data.

Do I need to pad binary numbers with leading zeros?

Yes, each hex digit must become exactly 4 binary digits. For example, hex "2" becomes "0010", not just "10". This maintains the correct place values.

About Octal Numbers

What is Octal?

Octal (base-8) uses digits 0-7. Each digit represents 3 bits of binary data. It's commonly used in Unix/Linux file permissions and some programming contexts.

Unix File Permissions

In chmod, each digit represents permissions: 4=read, 2=write, 1=execute. Sum them: 7=rwx, 6=rw-, 5=r-x, 4=r--, 3=wx, 2=w-, 1=x, 0=---.

Conversion Formula

Multiply each digit by 8 raised to its position power (from right, starting at 0). Example: 127 = (1×8²)+(2×8¹)+(7×8⁰) = 64+16+7 = 87

Common Questions

What does octal 755 mean in Unix?

755 = owner can read, write, execute (7); group can read and execute (5); others can read and execute (5). It's the standard permission for scripts and directories.

How do I convert octal to decimal manually?

Multiply each digit by 8 raised to its position power (starting from 0 on the right), then sum. For example, 127 = (1×64)+(2×8)+(7×1) = 64+16+7 = 87.

Why can't I use digits 8 or 9 in octal?

Octal is base-8, so it only uses digits 0-7. Digits 8 and 9 are invalid in octal numbers. If you see an 8 or 9, it's not a valid octal number.

What's the relationship between octal and binary?

Each octal digit represents exactly three binary digits (bits). For example, octal 7 (111), octal 5 (101). This makes octal useful for representing binary in a more human-readable form.
